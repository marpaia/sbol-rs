# sbol3-10205: Workflow stage role/type recommendation

**Status:** Deferred  •  **Spec section:** §A.1  •  **PolicyOptions knob:** `(none; coverage-only)`

## Spec wording

> An Identified object with a prov:wasGeneratedBy property referring to a prov:Activity with a child prov:Association that has a prov:hadRole property with a value from Table 20 should be of the corresponding type in Table 21.

## Ambiguity

The spec uses lowercase "should" without RFC-2119 force. No machine-checkable algorithm is given for cross-table lookup at validation time.

## Decision

Coverage records `Deferred(Policy)` until a Table 20/21 cross-reference policy is committed. No `PolicyOptions` knob is added.

## Rationale

The recommendation is workflow-scoped and depends on lab process. A validator can't resolve it without a registry.
