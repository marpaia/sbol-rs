# sbol3-12110: Static template Feature derivation count

**Status:** Partial  •  **Spec section:** §6.5  •  **PolicyOptions knob:** `derivation_provenance`

## Spec wording

> Each static Feature in the template MUST have exactly one derived Feature in each derived Component.

## Ambiguity

MUST-rule. "Each derived Component" is ambiguous when derivations chain.

## Decision

Default `Conservative`: warn on count mismatch. `Strict` errors. `Lenient` suppresses.

## Rationale

Same as sbol3-12109.

## Examples

### Conservative (default)
Matches today's validator behavior: the diagnostic shape and severity are exactly what the current test suite expects.

### Strict
(promotes the warning to an error, or activates a check that is currently suppressed.)

### Lenient
(drops the check; the rule is recorded in coverage as suppressed.)
