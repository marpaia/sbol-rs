# sbol3-12106: Derived Collection membership

**Status:** Partial  •  **Spec section:** §6.5  •  **PolicyOptions knob:** `derived_collection_membership`

## Spec wording

> Components derived from a CombinatorialDerivation SHOULD appear in a Collection that is referenced by the CombinatorialDerivation.

## Ambiguity

SHOULD-rule. Producers may write derived Components to disparate documents.

## Decision

Default `Conservative`: warn when the derived Component has no enclosing Collection in this document. `Strict` errors. `Lenient` suppresses.

## Rationale

Cross-document membership requires a resolver. Local default is best-effort.

## Examples

### Conservative (default)
Matches today's validator behavior: the diagnostic shape and severity are exactly what the current test suite expects.

### Strict
(promotes the warning to an error, or activates a check that is currently suppressed.)

### Lenient
(drops the check; the rule is recorded in coverage as suppressed.)
