# sbol3-10105: Unknown SBOL-namespace predicates

**Status:** Partial  •  **Spec section:** §5.2  •  **PolicyOptions knob:** `unknown_sbol_predicates`

## Spec wording

> The SBOL namespace MUST NOT be used for any entities or properties not defined in this specification.

## Ambiguity

The spec forbids unknown SBOL-namespace predicates but does not say how a tool should treat a document that contains them. Forward-compat needs to allow future SBOL revisions; conformance gates need to reject them.

## Decision

Default `Strict`: any SBOL-namespace predicate not in the catalog is an error. `Permissive` (`Lenient`) downgrades to warning so a stale tool can still parse a newer SBOL document.

## Rationale

The spec language is unambiguous (`MUST NOT`). Strict is the only setting that gates conformance; Permissive exists for forward-compat tooling only.

## Examples

### Conservative (default)
Matches today's validator behavior: the diagnostic shape and severity are exactly what the current test suite expects.

### Strict
(promotes the warning to an error, or activates a check that is currently suppressed.)

### Lenient
(drops the check; the rule is recorded in coverage as suppressed.)
