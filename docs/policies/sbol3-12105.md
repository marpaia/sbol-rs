# sbol3-12105: Derived Feature template provenance

**Status:** Partial  •  **Spec section:** §6.5  •  **PolicyOptions knob:** `derivation_provenance`

## Spec wording

> Each Feature of a derived Component SHOULD be derived (via prov:wasDerivedFrom) from a Feature of the template Component.

## Ambiguity

SHOULD-rule. Derived Components may include Features that the workflow added intentionally without template lineage.

## Decision

Default `Conservative`: warn when a derived Feature has no prov:wasDerivedFrom edge pointing into the template. `Strict` errors. `Lenient` suppresses.

## Rationale

Lineage tracking is informational by default; CI gates that enforce DBTL provenance can promote.

## Examples

### Conservative (default)
Matches today's validator behavior: the diagnostic shape and severity are exactly what the current test suite expects.

### Strict
(promotes the warning to an error, or activates a check that is currently suppressed.)

### Lenient
(drops the check; the rule is recorded in coverage as suppressed.)
