# sbol3-12107: Derived Component types inherit from template

**Status:** Partial  •  **Spec section:** §6.5  •  **PolicyOptions knob:** `derivation_provenance`

## Spec wording

> A derived Component SHOULD inherit type properties from its template Component.

## Ambiguity

SHOULD-rule. "Inherit" is not defined: union? Subset? Same set?

## Decision

Default `Conservative`: warn when the derived Component's `sbol:type` set is not a superset of the template's. `Strict` errors. `Lenient` suppresses.

## Rationale

Superset semantics match producer intuition (derived can specialize but not lose category).

## Examples

### Conservative (default)
Matches today's validator behavior: the diagnostic shape and severity are exactly what the current test suite expects.

### Strict
(promotes the warning to an error, or activates a check that is currently suppressed.)

### Lenient
(drops the check; the rule is recorded in coverage as suppressed.)
