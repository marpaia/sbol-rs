# sbol3-12109: Static template Feature property equality

**Status:** Partial  •  **Spec section:** §6.5  •  **PolicyOptions knob:** `derivation_provenance`

## Spec wording

> A static Feature in a derived Component MUST have the same properties as its template Feature.

## Ambiguity

MUST-rule. "Same" must be defined property-by-property; orientations and locations are part of the Feature.

## Decision

Default `Conservative`: compare core SBOL properties (orientation, role, type, locations) and emit warning on divergence. `Strict` errors. `Lenient` suppresses.

## Rationale

Warning-by-default matches sbol-rs's history of preferring iterative design diagnostics; CI gates promote.

## Examples

### Conservative (default)
Matches today's validator behavior: the diagnostic shape and severity are exactly what the current test suite expects.

### Strict
(promotes the warning to an error, or activates a check that is currently suppressed.)

### Lenient
(drops the check; the rule is recorded in coverage as suppressed.)
