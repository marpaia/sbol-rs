# sbol3-12111: Variable Feature cardinality enforcement

**Status:** Partial  •  **Spec section:** §6.5  •  **PolicyOptions knob:** `derivation_provenance`

## Spec wording

> A variable Feature's derived count MUST match its `cardinality` (zeroOrOne, exactlyOne, zeroOrMore, oneOrMore).

## Ambiguity

MUST-rule. The validator can compute the count locally.

## Decision

Default `Conservative`: warn on count outside the cardinality. `Strict` errors. `Lenient` suppresses.

## Rationale

Same as sbol3-12109.

## Examples

### Conservative (default)
Matches today's validator behavior: the diagnostic shape and severity are exactly what the current test suite expects.

### Strict
(promotes the warning to an error, or activates a check that is currently suppressed.)

### Lenient
(drops the check; the rule is recorded in coverage as suppressed.)
