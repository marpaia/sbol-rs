# sbol3-12112: Derived SubComponent variant lookup

**Status:** Partial  •  **Spec section:** §6.5  •  **PolicyOptions knob:** `derivation_provenance`

## Spec wording

> Each derived SubComponent's `instanceOf` MUST refer to a Component in the VariableFeature's variants, a member of a referenced variantCollection, or a derivation of a referenced variantDerivation.

## Ambiguity

MUST-rule. The lookup spans three indirection levels; failure could mean a missing variant or a mis-targeted derivation.

## Decision

Default `Conservative`: warn when the instanceOf doesn't resolve through any of the three sources. `Strict` errors. `Lenient` suppresses.

## Rationale

Cross-derivation membership requires a resolver. Local default is best-effort.

## Examples

### Conservative (default)
Matches today's validator behavior: the diagnostic shape and severity are exactly what the current test suite expects.

### Strict
(promotes the warning to an error, or activates a check that is currently suppressed.)

### Lenient
(drops the check; the rule is recorded in coverage as suppressed.)
