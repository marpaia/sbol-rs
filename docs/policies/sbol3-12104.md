# sbol3-12104: Empty CombinatorialDerivation template warning

**Status:** Partial  •  **Spec section:** §6.5  •  **PolicyOptions knob:** `derivation_provenance`

## Spec wording

> A CombinatorialDerivation's template Component SHOULD have at least one Feature.

## Ambiguity

SHOULD-rule with no force. The spec doesn't define what "at least one" means for variable Features specifically.

## Decision

Default `Conservative`: warn when the template has zero Features. `Strict` errors. `Lenient` suppresses.

## Rationale

An empty template is almost always a bug; a warning is the right default. Strict suits CI gates against design libraries.

## Examples

### Conservative (default)
Matches today's validator behavior: the diagnostic shape and severity are exactly what the current test suite expects.

### Strict
(promotes the warning to an error, or activates a check that is currently suppressed.)

### Lenient
(drops the check; the rule is recorded in coverage as suppressed.)
