# sbol3-12115: Type-determining referent resolution

**Status:** Partial  •  **Spec section:** §6.5  •  **PolicyOptions knob:** `derivation_provenance`

## Spec wording

> The types of a derived Feature SHOULD be determined by resolving the type-determining referent of its template, transitively.

## Ambiguity

SHOULD-rule. Resolution may require chasing prov:wasDerivedFrom across multiple objects.

## Decision

Default `Conservative`: warn when a derived Feature's types diverge from the resolved referent's types. `Strict` errors. `Lenient` suppresses.

## Rationale

Multi-hop resolution is expensive; default is best-effort with a warning.

## Examples

### Conservative (default)
Matches today's validator behavior: the diagnostic shape and severity are exactly what the current test suite expects.

### Strict
(promotes the warning to an error, or activates a check that is currently suppressed.)

### Lenient
(drops the check; the rule is recorded in coverage as suppressed.)
