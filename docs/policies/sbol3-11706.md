# sbol3-11706: Range/Cut Table 10 relations

**Status:** Partial  •  **Spec section:** §6.4.3  •  **PolicyOptions knob:** `conflict_resolution`

## Spec wording

> The restriction values of a Constraint over Features whose Locations are Ranges or Cuts MUST satisfy the relations listed in Table 10.

## Ambiguity

Table 10 is a closed lookup, but "satisfy" means the validator must compute relations like overlaps/precedes from the Locations themselves. Locations on different Sequences have undefined relation.

## Decision

Default `Conservative`: check only when both Locations are on the same Sequence. `Strict` errors if Sequences differ (treating the relation as vacuously violated). `Lenient` skips.

## Rationale

Cross-Sequence Range/Cut relations are not meaningful; the spec doesn't define them. Conservative is the only one that doesn't surprise the producer.

## Examples

### Conservative (default)
Matches today's validator behavior: the diagnostic shape and severity are exactly what the current test suite expects.

### Strict
(promotes the warning to an error, or activates a check that is currently suppressed.)

### Lenient
(drops the check; the rule is recorded in coverage as suppressed.)
